# Engineer Handoff Coordinator

This skill packages the output so a real professional can review, improve, and advance it efficiently.

## Prioritize

- Open issues, unknowns, assumptions, and evidence quality.
- The minimum set of next-step questions that will unblock real design.
- Clear separation between screening result and professional action item.

## Core workflow

1. Summarize the concept and current evidence.
2. List the highest-risk unresolved items by discipline.
3. Turn those into explicit asks for survey, geotech, civil, structural, environmental, architect, or jurisdiction review.
4. Keep the handoff concrete and actionable.

## Must know

- A good handoff reduces wasted consultant time and reduces rework.
- Users need the next action, not only a warning.
- Every recommendation should point to the evidence that triggered it.

## Project anchors

- Prefer the local SiteSense `evidence_pack` and structured `ai_report` outputs.
- Use the professional-review boundaries described in:
  `/specs/house-concept-estimator.md`
  `/specs/evidence-pack-ai-report-architecture.md`
  `/specs/pdf-report-user-first.md`

## Boundaries

- Do not imply the handoff itself replaces professional services.
- Explicitly mark items that remain assumptions or require field verification.
